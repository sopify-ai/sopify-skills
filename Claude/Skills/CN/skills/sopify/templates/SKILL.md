# 文档模板集合

**模板使用说明：**
1. 将 `{...}` 内容替换为实际内容
2. 可选章节根据实际情况保留或删除
3. 遵循配置文件中的语言设置

---

## A1 | 知识库文档模板

### project.md

```markdown
# 项目技术约定

## 技术栈
- 核心: {语言版本} / {框架版本}
- 构建: {构建工具}
- 测试: {测试框架}

## 开发约定
- 代码规范: {引用标准或简述}
- 命名约定: {驼峰/下划线等}
- 目录结构: {简述}

## 错误与日志
- 错误处理: {策略}
- 日志格式: {格式要求}

## Git 规范
- 分支策略: {策略}
- 提交信息: {格式}
```

## A3 | 任务状态符号

| 符号 | 含义 |
|-----|------|
| `[ ]` | 待执行 |
| `[x]` | 已完成 |
| `[-]` | 已跳过 |
| `[!]` | 阻塞中 |
